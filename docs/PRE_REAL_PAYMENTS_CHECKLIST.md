# Чеклист до первой реальной выплаты партнёру

**Для кого:** владелец / финансовый оператор.  
**Когда:** после Concierge Launch и успешной **симуляции** двух рельсов (`docs/CONCIERGE_LAUNCH_TREASURY_RUNBOOK.md` §12).  
**SSOT денег:** `ARCHITECTURAL_DECISIONS.md`, ADR-097, `lib/treasury/payout-rails.js`.

---

## 0. Симуляция (обязательно сначала)

- [ ] `npm run smoke:full-financial:rub` — **PASS** (RUB Direct)
- [ ] `npm run smoke:full-financial:intl` — **PASS** (KG / USDT)
- [ ] В `/admin/settings/finances`: **Drift ledger = OK**, нет критичного fiscal alert
- [ ] Тестовый пул по каждому рельсу: lock → CSV → ZIP → «Отметить как оплаченный» → акт в кабинете партнёра

## 8. Go / No-Go (одна строка)

**Go**, если: симуляция обоих рельсов PASS, нет drift/fiscal блокеров, первый пилотный пул ≤ согласованного лимита, все реквизиты сверены вручную.

**No-Go**, если: любой FAIL smoke, неопубликованные оферты, sandbox fiscal на prod, смешение рельсов в одном банковском платеже.
